# Code Review 规范

对当前代码变更进行审查，确保代码质量和竞赛要求。

## 审查维度

### 1. 代码质量
- 代码逻辑是否清晰
- 是否有明显的 Bug 或安全隐患
- 命名是否规范（变量、函数、类）
- 是否有重复代码可以提取
- 注释是否充分且有意义

### 2. 功能完整性
- 功能是否完整实现
- 是否有遗漏的边界条件
- 错误处理是否完善
- 是否有必要的输入验证

### 3. 性能考量
- 是否有性能瓶颈
- 是否有不必要的计算或 I/O
- 数据结构选择是否合理

### 4. 竞赛要求检查
- 是否引入了第三方依赖（需在 README 中说明）
- 是否有 API 密钥等敏感信息
- 代码是否为自主完成
- 是否有复用代码需要注明来源

## 输出格式

```
## Code Review 结果

### ✅ 通过项
- [x] 项1
- [x] 项2

### ⚠️ 建议改进
- 建议1: 说明
- 建议2: 说明

### ❌ 必须修复
- 问题1: 说明及修复建议
- 问题2: 说明及修复建议

### 📝 总结
整体评价和建议
```
