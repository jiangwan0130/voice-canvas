### 1.4 与同类方案的本质差异

典型的"语音→LLM→画图"方案是**文生图**（一次生成位图）。语绘是**指令式矢量绘图**：
- 每个图形是独立的矢量对象（有 id/label/坐标/样式），可单独编辑/删除/移动
- 逐笔动画生长，不是瞬间出现
- 手绘风格（彩铅排线），不是机械纯色填充
- 支持多轮对话编辑（"把树变大""在旁边画太阳""撤销"）

### 2.2 后端代码质量 (Python / FastAPI)

**✅ 优点：**
- `command_parser.py`：7 项校验层层递进，`COORD_KEYS` 用 `frozenset` 预计算，边界钳位覆盖 dx/dy 偏移量
- `json_repair.py`：三层修复管道可独立测试，括号栈逆向补全处理截断场景
- `local_rules.py`：长键优先排序防"清除"误吃"全部清除"，绘画名词检测防误杀
- `summarize_grid()`：注释清晰说明设计意图，减少 60-70% token 消耗
- 422 错误自定义 handler 记录请求体，方便调试
- Rate Limiter：ASR 20/min / generate 10/min / visual-verify 5/min，粒度合理
- Pydantic Model 完整定义 CanvasState / GridState / ConversationTurn

**⚠️ 可改进：**
- `llm_client.py` 的 SYSTEM_PROMPT 约 300 行内联字符串，建议拆分为模板文件
- `visual_verify()` 的请求体格式是 DashScope 风格，但调用的是 `LLM_API_BASE`（DeepSeek），两者 API 不一致
- 缺少统一错误码体系，目前用 HTTPException + 字符串
- `logging` 已正确使用（非 print），此项优于初版

### 2.3 前端代码质量 (React + TypeScript)

**✅ 优点：**
- TypeScript 类型定义完整（`commands.ts` 覆盖全部 ActionType/DrawObject/DrawInstruction）
- `renderer.ts`：逐笔动画核心用 `requestAnimationFrame`，矩形四边逐段生长、多边形逐边+闭合
- `ObjectStore`：Map 存储 O(1)，空间网格自动计算，snapshot/restore 深拷贝
- `HistoryManager`：双栈模式，redo 先存当前状态保证 undo→redo→undo 链完整
- `FuzzyMatcher`：四级降级，KNOWN_LABELS 与 system prompt 配方标签对齐
- `useVoice`：AnalyserNode VAD (RMS > 0.02) + MediaRecorder + Web Speech 备用，三重保护，含 cleanup 函数
- `CommandExecutor`：dispatch 模式按 action 分发，新增指令只需加 case
- `brush.ts`：策略模式，交叉排线(偏移 25°) + 线宽微扰(±0.25)模拟手握力道不均，细节到位
- `ErrorBoundary` 包裹 Canvas 防白屏

**⚠️ 可改进：**
- `renderer.ts` 722 行偏大，可拆分为 shapes/animations/controls 模块
- `CommandExecutor` 中 `update_object/move_object/delete_object` 的处理与 Renderer 中对应的 `console.warn` 分支形成双轨，数据流经过两层
- `brush.ts` 的 `sampleGradient` 用离屏 canvas + `getImageData` 采样，大量笔触时性能存疑（当前对象数少无感）
- `as any` 类型断言在 `CommandExecutor.ts` 中有几处，可用判别联合类型消除

### 3.3 推荐的 Demo 演示流程（3-5 分钟）

**第一阶段：开场 (30s)**
1. 打开应用 → 登录页展示品牌 + 水彩背景
2. 输入名字 → 进入画室
3. 口播："这是一个纯语音控制的绘图工具，接下来的所有操作都只用语音完成"

**第二阶段：首个场景 (1min)**
4. 点击🎤开始录音 → "画一棵绿色的树"
5. **亮点展示**：树干先画 → 树冠从下往上、从外向中心逐层生长（6-7 个圆依次出现）
6. TTS 播报"好的，在画布中央画了一棵树"

**第三阶段：多轮编辑 (1min)**
7. "在树的旁边画一个红色的太阳" → 太阳出现在右上
8. "把树变大一点" → 树整体放大（展示 update_object + 模糊匹配"树"）
9. "在太阳右边画一朵云" → 云朵出现在太阳右侧（展示空间推理"右边"）
10. "画一个房子，前面画一些花" → 展示由远及近分层绘制

**第四阶段：撤销演示 (30s)**
11. "撤销" → 花消失（<1ms 本地规则响应）
12. "重做" → 花恢复
13. 口播："注意撤销是瞬间完成的，因为它走的是本地规则引擎，不需要等待 AI"

**第五阶段：收尾 (30s)**
14. "清空画布" → 重新画一个场景（可选"画一只小鸟"展示复杂对象配方）
15. 点击导出按钮 → 下载 PNG
16. 总结："所有操作仅通过语音完成，从识别到渲染全链路闭环"

### 3.4 Demo 注意事项

- 🔴 **纯语音原则**：这是题目二的核心要求，Demo 中绝对不能出现鼠标点击画布或键盘输入。调试用的文本输入框应隐藏
- ✅ **展示速度对比**：撤销(本地规则 <1ms) vs 绘图(LLM 2-5s)，体现双路由价值
- ✅ **展示模糊匹配**："把那个红色的变大" → AI 定位到太阳，不是树
- ✅ **展示手绘风格**：镜头拉近画布，展示彩铅排线的交叉纹理效果
- ✅ **展示降级能力**：可提一句"如果网络不好，会自动切换浏览器内置语音识别"
- ⚠️ **避免冷场**：LLM 生成期间（2-5s）有状态指示器动画，不会空白
