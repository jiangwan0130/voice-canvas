## PR #7: 对象管理（本项目）

**目标**: 实现 ObjectStore（空间网格）、HistoryManager（撤销）、FuzzyMatcher（模糊匹配）

**依赖**: PR #5（脚手架就绪。可与 PR #6 并行）

**具体文件**:
```
新建:
  frontend/src/engine/ObjectStore.ts
    - Map<string, DrawObject> 存储
    - 3×2 空间网格索引
    - add / get / update / delete / getAll / clear / restore
    - getByGroup / findByLabel / findByColor / findByCell / getLast
    - toGridState(): 生成空间摘要 JSON

  frontend/src/engine/HistoryManager.ts
    - Snapshot 栈（max 50）
    - save(objects, action) / undo() / clear()

  frontend/src/engine/FuzzyMatcher.ts
    - findObject(text, store): 四级模糊匹配
    - Level 1: label 关键词
    - Level 2: 颜色（红→#ff, 蓝→#00f...）
    - Level 3: 位置（左→cell[0,0],[1,0]...）
    - Level 4: 最近创建
```

**验收标准**:
- ObjectStore.add() 返回的对象自动带 id + cellId
- 添加"太阳"到 (600,150) → cellId = "0,1"（中上）
- findByLabel("太阳") 返回匹配对象
- HistoryManager.save → undo → 恢复旧状态
- FuzzyMatcher: "把红色的变大" → 返回 fill 含 #ff 的对象

## PR #8: 后端核心链路（协作）

**目标**: 打通 语音→文本→路由→LLM→修复→校验→JSON 指令 全链路

**依赖**: PR #5

**分工**:

**芝士番薯负责**:
```
新建/重写:
  backend/asr.py
    - POST /api/asr 端点
    - 七牛云短语音听写（签名 + 调用）
    - 兜底: Web Speech API 文本透传

  backend/router.py
    - 指令路由: is_local_command(text) → local_rules / llm_client
    - 字符串长度哨兵: ≤12字 + 匹配关键词 → 本地

  backend/local_rules.py
    - 12 色映射表
    - 画笔切换: 彩铅/颜料笔
    - 画布控制: clear/undo
    - setColor/setWidth/setBrush 关键词匹配
```

**本项目负责**:
```
新建:
  backend/llm_client.py
    - DeepSeek v4 via 七牛云 MaaS
    - 构造 System Prompt（含空间网格 + last_action）
    - response_format: json_object
    - 超时 15s + 重试 1 次

  backend/json_repair.py
    - extract_json(raw): 正则去 markdown 包裹
    - repair_truncated(s): 括号栈补全
    - validate_and_fix(parsed): 字段退化

  backend/command_parser.py
    - action 白名单检查
    - 必填参数校验（缺→丢弃）
    - 可选参数默认值补全
    - 坐标边界钳位
    - 颜色格式校验
    - target 存在性检查
    - 逐条丢弃（不因一条非法丢整批）

修改:
  backend/main.py
    - POST /api/generate 端点
    - 调用路由器 → 返回 { instructions, reply, source }
```

**验收标准**:
- ASR: 上传音频 → 返回文本
- 规则: "换成红色" → 返回 setColor 指令 (~50ms)
- LLM: "画一朵花" → 返回完整 JSON 指令序列
- 修复: LLM 输出被 markdown 包裹 → 成功提取 JSON
- 校验: 非法 action → 丢弃，合法指令保留

## PR #9: 前端串联（本项目）

**目标**: 把所有前端模块串起来，实现完整交互链路

**依赖**: PR #6（渲染器）+ PR #7（对象管理）+ PR #8（后端）

**具体文件**:
```
新建/重写:
  frontend/src/hooks/useVoice.ts
    - MediaRecorder + AnalyserNode VAD
    - 静音 2s 截断 + 60s 上限
    - onResult 回调

  frontend/src/hooks/useSpeechFeedback.ts
    - window.speechSynthesis 封装
    - speak(text): 取消当前 → 新建 → 朗读

  frontend/src/services/api.ts
    - transcribe(audioBlob): POST /api/asr
    - generate(text, canvasState, lastAction): POST /api/generate

  frontend/src/engine/CommandExecutor.ts
    - 接收后端返回的 instructions + reply
    - 先保存快照到 HistoryManager
    - 逐条 dispatch:
      - 绘图指令 → renderer.executeInstruction(inst)
      - 编辑指令 → ObjectStore.update/move/delete
      - 控制指令 → undo/clear/speak
    - 完成后 TTS 朗读 reply

  frontend/src/App.tsx
    - 状态机: idle → recording → transcribing → generating → drawing → idle
    - 整合所有组件
```

**验收标准**:
- 语音→识别→指令→渲染→TTS 全链路打通
- 说"画一朵花" → Canvas 逐笔画出花
- 说"换成红色" → 后续绘图变红色
- 说"撤销" → 上一步消失
- 说"清除" → 画布归白

## PR #10: 对象编辑 + 修正闭环（本项目）

**目标**: 实现画完后的增量编辑和纯语音修正能力

**依赖**: PR #9（前端全链路）

**具体文件**:
```
修改:
  frontend/src/engine/CommandExecutor.ts
    - update_object: ObjectStore.update(target, params)
    - move_object: 根据对象类型计算偏移
    - delete_object: ObjectStore.delete(target)
    - 模糊匹配：target 不存在时调用 FuzzyMatcher

  frontend/src/App.tsx
    - lastActionRef: 每次响应后保存 {reply, instructions}
    - 发送请求时附带 last_action 上下文

  frontend/src/engine/ObjectStore.ts
    - 对象更新后重新计算 cellId
    - 编辑后触发全量重绘
```

**验收标准**:
- "在画布右边画一棵树" → 树出现在右侧
- "把树变大" → label 模糊匹配到树，增大 size
- "把树改成蓝色" → 模糊匹配到树，改色
- "太远了，往左移" → last_action 上下文定位到上次操作对象
- "撤销" → 回退修改
