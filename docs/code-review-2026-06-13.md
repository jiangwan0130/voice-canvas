## 详细分析

### 1. 🔴 七牛云 ASR 认证实现错误

**文件**: `backend/asr/qiniu_asr.py:41,57-75`

**当前实现**:
- Header: `Authorization: Bearer <token>`
- 签名: `HMAC-SHA1(secret_key, URL)` — 仅对 URL 签名

**七牛云 API 实际要求** (per developer.qiniu.com/dora/11869):
- Header: `Authorization: Qiniu <AccessKey>:<Sign>`
- 签名: `HMAC-SHA1(secret_key, method+path+host+content-type+\n\n+body)`

**影响**: 所有 ASR 请求被 401 拒绝 → 语音识别完全不可用。

**修复方向**: 使用七牛云官方 Python SDK 或在 `_gen_token` 中构造标准的 Qiniu 签名数据串。

---

### 2. 🔴 HTTPException 被 except Exception 吞没

**文件**: `backend/main.py:68-84`

```python
try:
    ...
    if not audio_data:
        raise HTTPException(400, "音频数据为空")  # ← 被下面的捕获吃掉了
    ...
except Exception as e:  # ← HTTPException is an Exception subclass!
    return {...}  # ← 返回 HTTP 200 而非 400
```

**影响**: 空文件验证返回 200 OK → 前端降级 Web Speech API → 掩盖真正的验证错误。

**修复方向**: 将 `raise HTTPException` 移到 `try` 块外，或在 `except` 中显式 `except HTTPException: raise`。

---

### 3. 🟠 复合指令首匹配即 return

**文件**: `backend/local_rules.py:79-81`

每个关键词匹配后直接 `return`，无指令累积能力。

**示例**: 「换成红色然后清除画布」→ CANVAS_KEYWORDS 先匹配到「清除」→ 返回 `[clear]` → 「换成红色」的颜色切换被丢弃。

**修复方向**: 改为累积模式 — 遍历所有规则类别，收集全部匹配指令。

---

### 4. 🟠 字典序导致多字键不可达

**文件**: `backend/local_rules.py:8,26,40`

| 输入 | 先匹配 | 被跳过的死代码 |
|------|--------|---------------|
| "红色" | "红" (颜色切换为**红**) | "红色" (颜色切换为**红色**) |
| "全部清除" | "清除" (好的，**清除**) | "全部清除" (好的，**全部清除**) |
| "用彩铅" | "彩铅" | "用彩铅"、"换成彩铅" |

**原因**: `keyword in text` substring 匹配 + Python dict 插入顺序。

**修复方向**: 将更长关键词放在 dict 前面，或改用等值比较。

---

### 5. 🟠 正则 `[设为]?` 语义错误

**文件**: `backend/local_rules.py:66`

`[设为]?` 是字符类（匹配 0-1 个"设"**或**"为"），非可选组。
应改为 `(?:设为)?`。

**影响**: 「线条粗细设为5」无法匹配 → 最自然的表述反而走到 LLM。

---

### 6. 🟡 "返回" 错误映射为 undo

**文件**: `backend/local_rules.py:33`

「返回」在中文 UI 中是导航操作（回到上一页），不是撤销绘图。
用户画完说「返回」→ 期待回到主菜单 → 实际撤销了最后一笔。

LLM 接上后会用 1200x800 生成坐标 → 前端 800x500 渲染 → 画面裁切。
